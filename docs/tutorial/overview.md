# The docolin walkthrough

!!! info "What you'll have at the end"
    A real guide of your own, published on docolin from a repository you control, carrying its first verified-working stamp, and readable by an AI agent. No prior git or command line needed, just a browser and an email address. About half an hour, unhurried.

**docolin** is an open commons of technical documentation: community-written guides that you and an AI read from the same place, each one carrying proof of whether its steps actually worked on real machines. It's the kind of docs you reach for instead of endure.

This walkthrough takes you from nothing to your own guide living on it, the whole arc, with Pango doing every step right alongside you. It's one path, top to bottom: follow the stops in order, each picks up where the last left off, and there's no command line anywhere.

!!! cards { cols=1 }
    - [1. Make a Codeberg account](./codeberg-account.md){ icon=user-round-plus horizontal }
      A free account on a community-run forge. This is where your guide's files live.

    - [2. Make your docolin account](./docolin-account.md){ icon=at-sign horizontal }
      Sign up and claim your handle, the name your work is credited to.

    - [3. Write your first guide](./first-guide.md){ icon=file-pen-line horizontal }
      Make a repository, then write your first guide as one plain Markdown file.

    - [4. Publish it on docolin](./go-live.md){ icon=upload horizontal }
      Connect the repo and watch your guide go live, callouts, attribution, and all.

Ready? Pango's already lacing up. [Start with the Codeberg account.](./codeberg-account.md)
